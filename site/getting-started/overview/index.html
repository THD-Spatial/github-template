    Overview¶

    The THD-Spatial GitHub Template provides a standardized foundation for creating open-source projects within our organization. This template ensures consistency, quality, and compliance with best practices across all repositories.

    Purpose¶

    This template was created to:

    • Streamline repository setup - Provide pre-configured files and structures
    • Ensure compliance - Meet all requirements for open-sourcing projects
    • Maintain consistency - Standardize documentation across the organization
    • Simplify onboarding - Help new contributors understand project requirements

    Who Should Use This Template?¶

    This template is for THD-Spatial group members who want to:

    • Create a new open-source project
    • Open-source an existing private project
    • Ensure their project meets organizational standards

    What Makes a Repository "Open Source Ready"?¶

    A repository is considered ready for open-sourcing when it:

    1. ✅ Has a proper LICENSE file
    2. ✅ Includes comprehensive documentation
    3. ✅ Provides contribution guidelines
    4. ✅ Follows a code of conduct
    5. ✅ Properly manages large files with Git LFS
    6. ✅ Contains no sensitive information
    7. ✅ Has been reviewed and tested

    Template Structure¶

    GitHub-Template/
├── .gitattributes           # Git LFS configuration
├── .gitignore              # Files to ignore
├── .readthedocs.yaml       # Read the Docs config
├── LICENSE                 # Project license
├── README.md               # Project overview
├── CONTRIBUTING.md         # Contribution guidelines
├── CODE_OF_CONDUCT.md      # Community standards
├── OPEN_SOURCE_CHECKLIST.md # Requirements checklist
├── docs/                   # Documentation source
└── mkdocs.yml             # Documentation config

    Key Features¶

    Pre-configured Files¶

    All essential files are included with placeholder content that you can customize for your specific project.

    Comprehensive Checklist¶

    The Open Source Checklist guides you through every requirement, ensuring nothing is missed.

    Documentation Setup¶

    Complete MkDocs and Read the Docs configuration for professional documentation hosting.

    Git LFS Integration¶

    Pre-configured .gitattributes for managing large data files commonly used in spatial research.

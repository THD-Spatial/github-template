    Code of Conduct¶

    A Code of Conduct establishes ground rules for behavior and creates a welcoming, inclusive environment for all contributors.

    Why a Code of Conduct Matters¶

    A Code of Conduct:

    • Creates safety - Defines acceptable behavior and consequences
    • Encourages participation - Makes everyone feel welcome
    • Sets standards - Establishes community expectations
    • Prevents issues - Proactively addresses potential problems
    • Provides recourse - Gives reporting procedures for violations

    Best Practice

    Even for small projects, having a Code of Conduct signals that you value a positive community environment.

    Contributor Covenant¶

    The Contributor Covenant is the most widely-used Code of Conduct in open source.

    Why Choose Contributor Covenant?¶

    • ✅ Industry standard (used by thousands of projects)
    • ✅ Well-tested and proven effective
    • ✅ Available in multiple languages
    • ✅ Regularly updated
    • ✅ Clear and comprehensive

    How to Add a Code of Conduct¶

    Step 1: Choose a Version¶

    Visit Contributor Covenant and select the latest version (currently 2.1).

    Step 2: Create the File¶

    1. Create CODE_OF_CONDUCT.md in your repository root
    2. Copy the full Contributor Covenant text
    3. Customize the contact information section

    Step 3: Customize Contact Information¶

    Update the enforcement section with your contact details:

    ## Enforcement

Instances of abusive, harassing, or otherwise unacceptable behavior may be
reported to the community leaders responsible for enforcement at
[INSERT CONTACT EMAIL HERE].

All complaints will be reviewed and investigated promptly and fairly.

    Replace with:

    ## Enforcement

Instances of abusive, harassing, or otherwise unacceptable behavior may be
reported to the community leaders responsible for enforcement at
THD-Spatial-conduct@example.com.

All complaints will be reviewed and investigated promptly and fairly.

    Step 4: Reference in README¶

    Add a link to your Code of Conduct in README.md:

    ## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). Please read it before contributing.

    Key Sections to Understand¶

    Our Pledge¶

    Defines the commitment to creating an open and welcoming environment.

    We as members, contributors, and leaders pledge to make participation in our
community a harassment-free experience for everyone, regardless of age, body
size, visible or invisible disability, ethnicity, sex characteristics, gender
identity and expression, level of experience, education, socio-economic status,
nationality, personal appearance, race, caste, color, religion, or sexual
identity and orientation.

    Our Standards¶

    Examples of Positive Behavior:¶

    • Using welcoming and inclusive language
    • Being respectful of differing viewpoints
    • Gracefully accepting constructive criticism
    • Focusing on what's best for the community
    • Showing empathy towards others

    Examples of Unacceptable Behavior:¶

    • Trolling, insulting/derogatory comments, personal attacks
    • Public or private harassment
    • Publishing others' private information without permission
    • Other conduct inappropriate in a professional setting

    Enforcement Responsibilities¶

    Defines maintainer responsibilities:

    • Clarifying and enforcing standards
    • Taking appropriate corrective action
    • Removing/rejecting contributions that violate the Code of Conduct

    Scope¶

    Applies to: - Project spaces (issues, pull requests, etc.) - Public spaces when representing the project - Official project communication channels

    Enforcement¶

    Describes the reporting process and consequences.

    Enforcement Guidelines¶

    The Contributor Covenant includes tiered consequences:

    1. Correction¶

    Impact: Minor violation or misunderstanding

    Consequence: Private warning with explanation, possibly public apology requested

    2. Warning¶

    Impact: Single incident or series of actions

    Consequence: Warning with consequences for continued behavior, temporary interaction ban

    3. Temporary Ban¶

    Impact: Serious violation or sustained inappropriate behavior

    Consequence: Temporary ban from interaction for specified period

    4. Permanent Ban¶

    Impact: Pattern of violations, harassment, or aggression

    Consequence: Permanent ban from project community

    Alternative Codes of Conduct¶

    While Contributor Covenant is recommended, alternatives exist:

    Setting Up Enforcement¶

    Designate Enforcers¶

    Decide who handles Code of Conduct issues:

    • Small projects: Repository owner(s)
    • Medium projects: Core maintainers
    • Large projects: Dedicated moderation team

    Create Contact Method¶

    Options for reporting violations:

    1. Email: Create dedicated email (e.g., conduct@project.org)
    2. Private channel: Discord/Slack private channel
    3. Platform: Use GitHub's moderation features
    4. Form: Create private reporting form

    Privacy

    Ensure reporting mechanisms maintain confidentiality to protect reporters.

    Document Process¶

    Create internal guidelines for handling reports:

    ## Internal Enforcement Process

1. **Receive report**: Acknowledge within 24 hours
2. **Investigate**: Gather information from all parties
3. **Decide**: Determine if violation occurred and severity
4. **Act**: Apply appropriate consequence
5. **Document**: Record incident and action taken
6. **Follow-up**: Check with reporter and monitor situation

    For Research Groups¶

    Research projects may want additional guidelines:

    ## Research Conduct

In addition to our Code of Conduct, we expect:

- **Data integrity**: No fabrication or falsification of data
- **Attribution**: Proper credit for contributions and ideas
- **Collaboration**: Respect for collaborative research norms
- **Confidentiality**: Respect for unpublished work and data
- **Authorship**: Fair and transparent authorship decisions

    Checklist¶

    Before finalizing your Code of Conduct:

    • [ ] CODE_OF_CONDUCT.md created in repository root
    • [ ] Latest Contributor Covenant version used (or appropriate alternative)
    • [ ] Contact information customized
    • [ ] Enforcement team designated
    • [ ] Reporting mechanism established
    • [ ] Internal enforcement process documented
    • [ ] Code of Conduct referenced in README
    • [ ] All maintainers familiar with enforcement process

    Common Questions¶

    Do small projects need a Code of Conduct?¶

    Yes. It signals professional standards and welcoming environment regardless of project size.

    What if we never get reports?¶

    That's usually good! Having a CoC prevents issues and sets expectations even if never invoked.

    Can we modify the Contributor Covenant?¶

    Yes, but only minor modifications are recommended. Major changes defeat the purpose of using a standard.

    Who enforces the Code of Conduct?¶

    Project maintainers, though larger projects may have dedicated moderation teams.

    What if a maintainer violates the Code?¶

    They should be held to the same standards. Consider having escalation path to organization leaders.
